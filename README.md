# cf-monitor

Self-contained Cloudflare account monitoring. One worker per account.

Error collection, feature budgets, circuit breakers, cost protection — with zero D1 dependencies.

## Why

Traditional monitoring SDKs require a central account to collect and process telemetry. `cf-monitor` is different: install it on any Cloudflare account, and it monitors everything on **that account only**. No central infrastructure, no cross-account HMAC, no complex forwarding.

## Features

- **Error Collection** — Tail worker captures errors from all workers, deduplicates, creates GitHub issues
- **Feature Budgets** — Per-feature daily/monthly resource limits with automatic circuit breakers
- **Circuit Breakers** — Feature, account, and global kill switches via KV
- **Cost Protection** — Per-invocation resource limits prevent runaway loops (prevents $5K billing incidents)
- **Gap Detection** — Identifies workers without monitoring coverage
- **Worker Discovery** — Auto-discovers all workers on the account via CF API
- **Slack Alerts** — Budget warnings at 70%/90%/100%, error notifications
- **Analytics Engine** — All metrics stored in AE (90-day retention, SQL queries, 100M writes/month free)

### Optional (AI-powered)

- Pattern Discovery — AI-assisted transient error detection
- Health Reports — Natural language account health summaries
- Coverage Auditor — SDK integration quality scoring

## Quick Start

```bash
# 1. Install
npm install @littlebearapps/cf-monitor

# 2. Provision resources (KV namespace, AE dataset)
npx cf-monitor init --account-id YOUR_ACCOUNT_ID

# 3. Deploy the monitor worker
npx cf-monitor deploy

# 4. Wire all workers as tail consumers
npx cf-monitor wire --apply
```

## SDK Usage

Wrap your worker with `monitor()` — everything else is auto-detected:

```typescript
import { monitor } from '@littlebearapps/cf-monitor';

export default monitor({
  fetch: async (request, env, ctx) => {
    const data = await env.DB.prepare('SELECT * FROM items LIMIT 100').all();
    return Response.json(data);
  },
  scheduled: async (event, env, ctx) => {
    await env.KV.put('last-run', new Date().toISOString());
  },
});
```

**Required bindings** in your worker's wrangler config:

```jsonc
{
  "kv_namespaces": [
    { "binding": "CF_MONITOR_KV", "id": "YOUR_KV_ID" }
  ],
  "analytics_engine_datasets": [
    { "binding": "CF_MONITOR_AE", "dataset": "cf-monitor" }
  ]
}
```

## Architecture

```
Your Workers ──(tail)──> cf-monitor worker ──> GitHub Issues
    │                        │                    Slack Alerts
    │                        │
    └──(AE write)──> Analytics Engine <──(SQL)── cf-monitor crons
                                                  (budgets, gaps,
                                                   discovery, health)
```

One worker handles everything:

| Handler | Purpose |
|---------|---------|
| `tail()` | Error capture from all tailed workers |
| `scheduled()` | Cron multiplexer: metrics, budgets, gaps, discovery |
| `fetch()` | Status API: `/status`, `/errors`, `/budgets`, `/workers` |

## Storage

- **Analytics Engine** — All metrics and telemetry (zero D1)
- **KV** — Circuit breaker state, budget config, error dedup, worker registry

## Configuration

See `cf-monitor.yaml` generated by `npx cf-monitor init`.

## Migrating from @littlebearapps/platform-consumer-sdk

```typescript
// Before
import { platformWorker } from '@littlebearapps/platform-consumer-sdk/worker';
export default platformWorker({ project: 'scout', fetch: handler });

// After
import { monitor } from '@littlebearapps/cf-monitor';
export default monitor({ fetch: handler });
```

Update wrangler bindings: `PLATFORM_CACHE` -> `CF_MONITOR_KV`, `PLATFORM_ANALYTICS` -> `CF_MONITOR_AE`, remove `PLATFORM_TELEMETRY` queue.
